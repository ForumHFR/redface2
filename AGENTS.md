# AGENTS.md

## Project

Android client for forum.hardware.fr — currently in **specification phase** (no application code).
All documentation lives in `docs/` (Jekyll + just-the-docs, deployed on GitHub Pages).

## Setup

```bash
# No build yet — spec phase only
# Jekyll preview (requires Ruby + Bundler)
cd docs && bundle install && bundle exec jekyll serve
```

## Structure

```
docs/
  index.md           # Homepage, overview diagram
  stack.md           # Technical stack decisions
  architecture.md    # Gradle modules, layers, data flow, session, errors
  navigation.md      # Screens, deep linking, back stack
  models.md          # Kotlin data classes (source of truth for types)
  mvi.md             # MVI pattern, ViewModel/Screen examples
  features.md        # Community features, extension architecture
  naming.md          # App naming candidates
  roadmap.md         # Development phases
  contributing.md    # Conventions, tests, a11y, i18n
  _config.yml        # Jekyll config (spec version in footer)
```

## Code style

- Kotlin code examples must be conceptually compilable
- English for code, French for documentation and issues
- Conventional Commits: `feat:`, `fix:`, `docs:`, `chore:`, `test:`

## Testing

No tests yet (spec phase). Test strategy defined in `docs/contributing.md`:
- JUnit 4 + MockK + Robolectric + Turbine
- 100% coverage on parser, database, ViewModels
- HTML fixtures captured from real HFR pages, never fabricated

## Spec consistency rules

After modifying any spec file, check all related files:
- `models.md` changes → verify `mvi.md`, `architecture.md`, `navigation.md`
- `architecture.md` changes → verify `features.md`, `contributing.md`, `index.md`
- Mermaid diagrams must match surrounding text
- Module names must be identical across all files

## PR and commit conventions

- All AI-assisted commits must include `Co-Authored-By: {exact model name} <noreply@anthropic.com>`
- Never close an issue without an explanatory comment
- Issues and comments generated by AI must state who requested the action

## Project-specific

- `numreponse` is unique **per category**, not globally — primary key is `(cat, numreponse)`
- HFR BBCode: `[fixed]` is block-level only. No `[size=X]` tag. Color closes with `[/#XXXXXX]`.
- Prefetch must use unauthenticated requests to avoid marking flags as read
- Compose Navigation does not support URI fragments (`#t{id}`) — requires custom handling
